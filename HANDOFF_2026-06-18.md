## 1. Portfolio Backtest Engine Rewrite

### Problem
The old engine (`portfolio_backtest.py`) replayed Stage 4 pre-computed trade outcomes and scaled PnL by `equity / stage4_equity` ratio. Using **current equity** for sizing produced exponential compounding: $10K → $5.92 quadrillion over 2,461 trades.

### Solution
Complete rewrite of `apps/worker/src/quant_terminal_worker/stage4/portfolio_backtest.py`:

**What it does now:**
- Walks forward **candle by candle** across a unified timeline (union of all asset candle timestamps)
- For each candle: manages existing positions (TP/SL/pyramid/hard-exit) then processes new signals
- Uses Stage 4 setup parameters only (TP/SL %, pyramid config, leverage, hard exit hours, fees) — never trade outcomes
- Loads fresh signals from canonical Stage 1 scores + repository signal packets
- Evaluates TP/SL/pyramid against **actual candle OHLC prices** per asset
- Enforces shared margin: `free_margin = equity − sum(all_open_position_margins)`
- Enforces one position per asset at signal processing time
- Computes PnL from actual price movement × position size

**API change:**
- `run_portfolio_backtest()` now requires `repository` parameter for loading signals and candles
- Updated callers: `main.py` line 1188, `jobs.py` line 451

**Tests:**
- Rewrote `tests/test_portfolio_backtest.py` with mock repository and candle reader
- Updated `tests/test_api.py::test_portfolio_backtest_endpoint_writes_pool_artifacts`
- All 5 portfolio tests pass. 9 pre-existing failures in `test_execution_wake_runner.py` (unrelated)

### Remaining Concerns
- **Sizing basis**: Currently uses `current_equity` (intentional — matches real trading behavior). If results still show unrealistic growth, switch to `initial_capital_usdt`.
- **Margin allocation sum**: With 6 assets at 25-30% each (150-180% total), most entries will be skipped for insufficient margin. This is correct behavior but may produce high skip rates.
- **Signal cursor advancement**: While a position is open, signals for that asset are silently advanced past. These don't appear in skipped_signals. Should they?
- **Pyramid look-ahead bias**: Fixed from the old engine by evaluating pyramid step triggers per candle.
