### Wake Runner Test Fixes (570bdc6)
- Fixed 9 pre-existing failures in test_execution_wake_runner.py
- Root cause: test fixtures set `margin_allocation_pct` and `leverage` on the route object but omitted `manual_sizing_enabled: True`
- Without that flag, `_route_sizing_policy()` in wake_runner.py ignores route-level sizing and falls back to bundle sizing
- The test bundles only had pyramid config (step_pct, max_legs) with no sizing keys, producing zeros for margin/leverage
- This blocked all pyramid trigger evaluation (eligible=False, trigger_reached=False)
- Fix: added `"manual_sizing_enabled": True` to all 9 route definitions via replace_all
- All 130 tests now pass (35 wake runner + 95 API + 4 portfolio backtest)
