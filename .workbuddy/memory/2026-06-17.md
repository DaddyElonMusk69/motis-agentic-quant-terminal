## Stage 1A Strategy Optimization - iter_003_v0.1 → v0.4

**Session:** sui-vegas_ema_recursive_features-strategy-v01, iteration iter_003_v0.1

**Baseline (v0.3):**
- Total: 1273, Failures: 673 (52.9%), Mismatches: 453, Neutral: 220, Protected: 600
- Net regression from v0.2: failures +44, neutral +82, protected -44
- The 1D momentum SKIP veto was too aggressive — created 82 additional NEUTRALs while only fixing 38 mismatches

**Root cause:** v0.3's approach of SKIPping when 1D opposes a slim vote was too conservative. Converting ENTER→SKIP is always worse than a wrong-direction ENTER (SKIP is a guaranteed missed opportunity; wrong ENTER has a chance of being right in the future). Plus some of those SKIPped cases had truth matching the 1D direction.

**Changes made (v0.3 → v0.4):**

1. **Replaced SKIP veto with direction override**
   - Old: When vote margin ≤1 and 1D strongly opposes → SKIP (reason: `one_day_momentum_consensus_veto`)
   - New: When vote margin ≤1 and 1D strongly opposes → ENTER in 1D direction (reason: `one_day_trend_overrides_slim_vote_{long|short}`)
   - Confidence: 0.52 (moderate — better than guessing but less than consensus)
   - Rationale: If the daily trend is clearly established, it should override a slim-majority vote, not cause us to sit out entirely.

2. **Removed 1D tiebreak override entirely**
   - Removed `one_day_momentum_overrides_tiebreak` reason code
   - Restored original v0.2 tiebreak behavior: base_vote tiebreak resolves directly without 1D cross-check
   - Rationale: The tiebreak override was the biggest source of NEUTRAL increase. When weighted votes can't resolve, the base 2-vote minimum majority is already a reasonable signal.

3. **Preserved `_strong_momentum_direction` helper** - Still used by the direction override rule with ±6% threshold.

**Strategy version:** v0.3 → v0.4

**Expected effect:**
- Neutral count should drop significantly (the two SKIP-producing rules are removed)
- Mismatch count may change: direction override could turn some old-SKIPs into MATCH or MISMATCH
- Protected cases from v0.2 should be restored (tiebreak override was likely regressing some)

## Stage 1A Strategy Optimization - iter_004_v0.1 → v0.5

**Session:** sui-vegas_ema_recursive_features-strategy-v01, iteration iter_004_v0.1

**Baseline (v0.4):**
- Total: 1273, Failures: 641 (50.4%), Mismatches: 505, Neutral: 136, Protected: 632
- Net improvement from v0.3: failures -32, neutral -84. But still worse than v0.2 (629→641).
- New failure pattern: `one_day_trend_overrides_slim_vote` producing MISMATCHES (1D trend is too slow to react to short-term regime changes)

**Root cause analysis across all versions:**
The core unsolved problem is `feature_aligned_recursive_vegas` — all votes agree on one direction but truth is opposite. This happens ~40% of the time and no incremental rule has fixed it. The 7-vote system includes two noisy 5m-based components (`_interaction_direction` and `_ema_chart_direction`) that push the vote toward consensus too easily, creating false conviction.

**Changes made (v0.4 → v0.5):**

1. **Removed noisy 5m votes from the voting system**
   - Removed: `_interaction_direction` (price-EMA proximity on 5m) and `_ema_chart_direction` (5m chart trend)
   - Kept: 4 base votes + `context_chart_direction` (2h+1d chart agreement) = 5 votes total
   - Still requires ≥2 for weighted consensus
   - Rationale: The two removed votes are both derived from 5-minute data — the noisiest timeframe. They frequently push votes into false agreement when the actual directional move disagrees. The `context_chart_direction` already provides multi-timeframe confirmation via 2h+1d charts, making the 5m votes redundant and harmful.

2. **Replaced 1D momentum override with context chart override**
   - Old: When vote margin ≤1 and 1D momentum opposes → ENTER in 1D direction
   - New: When vote margin ≤1 and 2h+1d context chart opposes → ENTER in context chart direction
   - Reason code: `context_chart_overrides_slim_vote_{long|short}`
   - Rationale: The context chart (2h+1d agreement) is more responsive and independent than 1D momentum — it derives from actual price chart trends, not from the same feature momentum calculations that the vote already uses.

3. **Lowered voting thresholds to reduce NEUTRALs**
   - 2h momentum: 0.25% → 0.15%
   - 1d momentum: 0.75% → 0.50%
   - 5m micro_flow: 0.02% → 0.01%
   - Rationale: With 2 fewer votes in the system, each vote matters more. Lowering thresholds helps maintain enough directional signals to resolve.

**Strategy version:** v0.4 → v0.5

**Expected effect:**
- `feature_aligned_recursive_vegas` failures should decrease (fewer 5m-noise-driven false consensuses)
- Neutral count should decrease (lower thresholds + fewer votes needed for consensus)
- Mismatch patterns from `one_day_trend_overrides` replaced with `context_chart_overrides` — unknown effect
- The voting system is fundamentally simpler and cleaner
