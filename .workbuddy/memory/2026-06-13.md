## BNB iter_003 v0.1 → v0.4: Narrow weak_macro guard to macro-in-majority only

### Analysis
- v0.3 weak_macro_2_1_majority guard incorrectly applies to ALL 3-vote situations (3-0 unanimous + 2-1 splits) when |1d| < 2.0%
- Data shows: 2-1 with macro in MINORITY (micro+local agree vs weak macro) would have 57.1% DA if entered — better than overall 53.7%
- 2-1 with macro in MAJORITY (macro+one-LTF agree) would have only 43.9% DA → correctly blocked
- 3-0 unanimous with weak macro: 38.2% DA → correctly blocked

### v0.4 Change
- Added `and macro_direction == majority` to the guard condition (line 129)
- Only blocks when macro is among the agreeing votes (3-0 or 2-1 with macro on winning side)
- When micro+local agree against weak macro → ENTER with majority

### Expected Impact
- MATCH: 1017 → ~1049 (+32)
- MISMATCH: 876 → ~900 (+24)
- NEUTRAL failures: 300 → ~244 (-56)
- DA: 53.72% → ~53.82% (+0.1pp)
- Skip rate reduced by 56 entries
- Protected cases: unaffected

### Implementation
- STRATEGY_VERSION: v0.3 → v0.4
- Single condition change: `if len(directions) == 3:` → `if len(directions) == 3 and macro_direction == majority:`

## XAU iter_003 v0.3 → v0.4: Weak conflict resolution with daily_range

### Analysis
- XAU v0.3 DA: 47.60% (1425 MATCH / 2994 scored). Architecture differs from BNB: uses cross-timeframe conflict resolution + daily_range tiebreaker.
- Key failure: strong_multi_timeframe_consensus (816 MISMATCH) — XAU multi-timeframe consensus often wrong; conflict_resolved (289 MISMATCH) mostly macro wins but wrong.
- weak_cross_timeframe_conflict (73 NEUTRAL): when |1d|<1.0% and |2h|<1.0% and macro≠local, v0.3 SKIP'd. Data shows entering with daily_range yields 60.8% DA (31 MATCH, 20 MISMATCH) — better than overall.

### v0.4 Change
- In the weak conflict block (both returns < 1.0%), added a daily_range fallback:
  - If daily_range_direction exists → ENTER with range (new reason_code: `weak_conflict_resolved_by_range`)
  - If no range → SKIP (original `weak_cross_timeframe_conflict`, unchanged)
- No changes to other code paths

### Expected Impact
- MATCH: 1425 → ~1456 (+31)
- MISMATCH: 1569 → ~1589 (+20)
- NEUTRAL: 147 → ~74 (-73, 15 to null-truth ENTERs)
- DA: 47.60% → ~47.82% (+0.22pp)
- Skip rate reduced by ~51 entries (73 NEUTRAL → 15 null-truth ENTER + 7 no-range SKIP = 51 less SKIPs)

### Implementation
- STRATEGY_VERSION: v0.3 → v0.4
- Added `if daily_range_direction:` block before the existing SKIP return
- Comment updated: v0.2 → v0.4
